# セキュリティ

## 秘密情報

- API キー、パスワード、トークン等をソースコードにハードコードしない
- 環境変数またはシークレットマネージャを使用する
- `.env` ファイルは `.gitignore` に含める
- 必要な秘密情報が環境変数に設定されているか起動時に検証する
- 秘密情報がコミット履歴に混入した場合はその旨を報告し、ローテーションを促す

## 認証・認可

- 認証が必要なエンドポイントやリソースには必ず認証チェックを入れる
- 認可（権限）チェックを認証とは別に実施する
- 認証トークンの有効期限と失効処理を実装する

## 入力バリデーション

- 外部入力（ユーザー入力、API レスポンス、ファイル読み込み）は信頼しない
- SQL はパラメータ化クエリを使用する
- HTML 出力はサニタイズする
- コマンド実行にユーザー入力を直接渡さない

## エラーハンドリング

- エラーメッセージに内部情報（スタックトレース、DB構造、ファイルパス等）を含めない
- ログには記録し、ユーザーには汎用メッセージを返す

## 依存関係

- 既知の脆弱性がある依存関係を追加しない
- メンテナンスされていないパッケージの採用は避ける
