# 概要

指定された機能IDをもとに、Java アプリケーションのログから該当するスタックトレースを取得する。

## ローカル設定

スキル実行時は最初に `config.local.md` を読み込む。
ファイルが存在しない場合は `config.example.md` を参照してセットアップを案内する。

## 手順

### 1. ログファイルの特定

ユーザーからログディレクトリ・AP種別・機能IDを受け取る。
`config.local.md` のAP種別とファイル名パターンの対応を参照し、該当するログファイルを特定する。
ログローテーションにより同一AP種別のファイルが複数存在する場合はすべてを対象とする。

### 2. スタックトレースの抽出

ERRORレベルのログエントリからスタックトレースを完全に取得し、
その中にユーザー指定の機能IDが含まれているものだけを抽出する。

- `\tat ...`（スタックフレーム）、`Caused by: ...`（原因チェーン）、`\t... N more`（省略フレーム）をすべて含める
- 終端は次のログエントリ（タイムスタンプ行やログレベル行）の開始またはファイル末尾で判定
- 機能IDの照合はエラー行だけでなくスタックトレース全体を対象とする

### 3. 結果のファイル出力

抽出したエラーはファイルに出力する。ユーザーにはファイルパスと抽出件数のサマリーのみ伝える。

- 出力先: `./output/extract-stacktrace/stacktrace-<YYYYMMDD-HHMMSS>.md`（プロジェクトルート相対）
- エラー行 + 完全なスタックトレースをセットで記載
- 複数件ある場合は時系列順にすべて記載する（グルーピングや省略はしない）
