# codebase-scan 対象リポジトリ

このファイルを `repos.local.md` にコピーして、自社環境に合わせて記入してください。
`repos.local.md` は gitignore されているためコミットされません。

```
cp repos.example.md repos.local.md
```

## 対象リポジトリ

調査対象を以下のテーブルに列挙する。`パス` 列は実在する絶対パスに書き換えること。

| 名前 | パス            | 主な対象拡張子    | テスト除外パターン        | 備考         |
| ---- | --------------- | ----------------- | ------------------------- | ------------ |
| a    | /path/to/repo-a | `*.ts`, `*.tsx`   | `__tests__`, `*.test.*`   | フロント     |
| b    | /path/to/repo-b | `*.java`, `*.xml` | `*Test.java`, `src/test/` | バックエンド |
| c    | /path/to/repo-c | `*.go`            | `*_test.go`               | バッチ       |

列の意味:

- `名前` — TSV 出力の `repo` 列に入る短い識別子（英数字推奨）
- `パス` — リポジトリのルート絶対パス
- `主な対象拡張子` — `--include` のデフォルト候補（承認時に絞り込み可能）
- `テスト除外パターン` — テスト除外オプション選択時に `--exclude-dir` / `--exclude` に追加するパターン
- `備考` — 用途メモ（任意）
